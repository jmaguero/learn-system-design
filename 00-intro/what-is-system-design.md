# What is system design?

## What it is
System design is the practice of choosing components, interfaces, and data flows to meet requirements under real constraints (latency, cost, reliability, security).

## What you produce
A clear architecture, APIs, data model, scaling plan, and a list of trade-offs.

## Practice prompts
- Describe the difference between system design and coding a feature.
- List 5 non-functional requirements you would ask for upfront.
